      반영 전 검증

      정해진 양식의 엑셀 또는 CSV 파일을 먼저 검사합니다. 로컬·서버 검증을 모두 통과한 뒤 관리자가 마지막으로 확인해야 실제 DB에 반영됩니다.

      1. 양식 회원, 회원권, 결제, 정규시간을 한 줄에 정리
      2. 검증 필수값, 횟수 계산, 숫자, 코치명을 확인
      3. 확정 관리자 확인 후 서버에서 Supabase에 반영

      데이터 내보내기

      관리자 전용

      운영 중 필요한 회원, 회원권, 결제, 시간표, 상품, 코치 데이터를 엑셀/CSV/JSON으로 내려받는 화면입니다. 실제 서비스에서는 관리자 권한과 내보내기 이력을 남깁니다.

      1단계 기준

      실제 서비스로 넘어가기 전 DB, 권한, 결제, 알림, 기존자료 이전 준비 상태를 확인합니다.

      결제

      실결제 연결 설정

      PortOne 연결값은 이 브라우저에만 저장해서 데모 결제창 연결을 확인합니다. 실제 회원권 충전은 서버 검증과 웹훅 연결 후 자동화합니다.

      서버 DB

      실제 DB 읽기 상태

      로컬 설정값으로 Supabase에 접속해 주요 테이블을 읽을 수 있는지 확인합니다. 권한이 막히면 로그인 연결 단계가 필요하다는 뜻입니다.

      로그인

      로그인 제공자 상태

      통합앱 로그인에 필요한 Kakao, Naver Provider가 Supabase에서 켜져 있는지 읽기 전용으로 확인합니다.

      다음 설정 위치
      • Supabase Authentication > Providers에서 Kakao, Naver를 켭니다.
      • Redirect URL에는 통합앱의 회원 화면 주소를 등록합니다.
      • 첫 로그인 후 Authentication > Users의 UUID를 tn_users.auth_user_id에 연결합니다.
      http://127.0.0.1:8773/90-Dashboard/tennis-note-member-app/index.html

      권한

      권한 기준

      실사용 전 필수
      • 관리자: 전체 회원, 시간표, 결제/정산, 코치 권한, 정책 설정 관리
      • 코치: 같은 지점 시간표와 회원 수업기록 확인, 본인/대타 수업 처리
      • 회원: 본인 시간표, 운동일지, 회원권, 결제 요청, 커리큘럼만 확인
      • 서비스 키, 원본 매출, 개인정보 원본은 브라우저 앱에 넣지 않음

      회원권 상품

      할인권

      할인권은 관리자가 만들고, 코치는 관리자가 지급한 수량 안에서만 사용하게 두는 방식이 가장 안전합니다.

      코치 사전 등록

      관리자가 코치 이름과 휴대전화를 먼저 등록합니다. 코치가 같은 번호가 확인된 소셜 계정으로 로그인하면 코치 모드가 자동으로 연결됩니다.

      운영시간·브레이크

      관리자에서 정한 운영시간, 브레이크타임, 코치별 근무 블록이 통합앱의 회원 화면과 코치 모드에 같은 기준으로 반영됩니다.
